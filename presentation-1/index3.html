Styling components in core for all brands via variables

  • Limits the design
    • Business may not be happy. 'We can't do this design' doesn't sound very good. (eg: maximizer)
    • Dssigners won't be happy
  • CSS developers won't be happy
  • Extra layer of complexity
  • Lots of overides
  • Naive to expect that core can make changes to markup without brands having to refactor

Alternative:

  1. CORE CSS will contain variables for site settings (eg theme colors etc), scafolding (eg normalize), architecture (CSS 'objects' for layout ) and typography
  2. CSS style guide which must be refined and agreed upon and thoroughly understood by CSS devs
  3. CORE writes component markup (according to HTML style-guide) and CORE-specific CSS for components (according to CSS style-guide)
  4. CORE component CSS will be passed on to brands as boilerplate(ish) code which may be edited freely by devs.
  5. Pull requests to component CSS by brands will be examined to make sure it complies with style-guide requirements
